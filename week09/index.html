Notes

  • The Browser Object Model is a collection of properties and methods that contain information about the browser and computer screen.
  • Global variables can be accessed in all parts of the program and are properties of a global object.
  • Using popups is not recommended, you have to look for better alternatives.
  • The Screen object has more uses for mobile devices. It also allows you to do things like turn off the device's screen, detect a change in its orientation or lock it in a specific orientation.
  • Cookies are small files that are saved locally on a user's computer and can be used to store information that can then be retrieved between requests.
  • Cookies have many limitations, it is preferable to use LocalStorage, although cookies are still used, such as to keep a session open in a browser.
  • To remove a cookie, you need to set it to expire at a time in the past.
  • The window object provides some useful methods for scheduling the execution of a function, and for repeatedly executing functions at regular intervals.
  • setTimeout() method can be used to invoke a function after a set amount of time.
  • setInterval() method can be used to repeatedly invoke a function.
  • requestAnimationFrame() method can be used to produce smooth and animation optimized by utilizing the browser's built-in graphics capabilities.
  • A useful site that checks to see if a specific feature can be used is caniuse.com
  • The data- attribute is a way of embedding data in a web page using custom attributes that are ignored by the browser.
  • Is best practice to use feature detection before using any of the HTML5 API methods.
  • The Geolocation API is used to obtain the geographical position of the device.
  • Web workers allow us to do work in the background while the javascript is running. Since javascript only runs one process at a time.
  • The Service Worker API allows a worker script to run in the background with the added benefit of being able to intercept network requests.
  • Websocket is a new protocol that allows two-way communication with a server also known as push messaging. This means that a connection is kept open and responses are "pushed" to the client as soon as they are received.
  • wss:// is the secure protocol used by websockets.
  • The notification API is a good tool to communicate things to users, since a pop up will appear in some part of the screen, regardless of whether the user is not on our website.
  • HTML5 introduced the audio and video tags used to insert audio and video clips into a web page. It also introduced a Media API for controlling the playback of the clips using JavaScript.
  • All video or audio methods are available when the media file is completely downloaded. So it's appropriate to use the loadedmetadata event, which we can use to fire a function when all the video's metadata has loaded.
  • The canvas element was introduced to allow graphics to be drawn onto a web page in real time using JavaScript
  • To make sure that the browsers used by our users are compatible with the HTML5 APIs that we use, we can use the Shims and Polyfills libraries, which allow us to use the html 5 apis and add the necessary code to make them work in browsers not yet compatible.
