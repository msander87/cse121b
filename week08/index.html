Notes

  • The CSS3 transform property lets you lets you translate, rotate, scale, and/or skew any element on the page.
  • Translation functions allow you to move elements left, right, up, or down.
  • To move an element only vertically or horizontally you can use translatey or translatex.
  • The scale(x,y) function scales an element by the defined factors horizontally then vertically.
  • If we want to use several transforms functions we must do it in the same declaration, since declaring tranform again would overwrite the first one.
  • The rotate() function rotates an element around the point of origin by a specified angle value.
  • The skew(x,y) function specifies a skew along the x and y axes. As you would expect, the x specifies the skew on the x axis, and the y specifies the skew on the y axis.
  • The order in which we use the transform functions can return different results.
  • Using transitions in css is much simpler, runs faster than using javascript, and allows you to make subtle changes instead of making them abrupt.
  • To create a transition, we must think about what we want to achieve as the final effect and add that property with a origin value, otherwise the transition function will not work.
  • The transition-duration property sets how long the transition will take.
  • The transition-timing-function allows us to create a much more modern animation, it allows us to give more life to the elements, giving them a more fluid and not so mechanical movement.
  • The transition-delay function will make the transitions not start immediately. It is recommended when we do not want everything to happen immediately, such as an unintentional touch on the screen of a cell phone, with this we make sure that the user does want to touch an element since he presses it with more intention.
  • We can use all 4 transition functions faster by just writing the word transition: and then putting the value for each transition on the same line.
  • Animation is a new feature that allows us to control the entire animation process through keyframes.
  • The animation-name property is used to attach an animation to an element.
  • The animation-duration property defines the length of time (in seconds or milliseconds) an animation takes to complete one iteration.
  • The animation-timing-function is similar to the transition-timing-function property and will make the animation smoother and more subtle.
  • The animation-iteration-count property lets you define how many times the animation will play through.
  • Animation-direction is used to change the orientation in which the animation runs.
  • The animation-fill-mode property defines what happensbefore the first animation iteration begins and after the last animation iteration concludes.
  • The animation-play-state property defines whether the animation is running or paused.
  • Canvas is an API, we can draw shapes and lines, arcs and text, gradients and patterns. It gives us the power to manipulate pixels in images and even video, all this through javascript.
  • All drawing on the canvas happens via the Canvas JavaScript API.
  • With Canvas we can only create rectangles and squares, but we can create other shapes by using paths to create a plane for the lines, arcs and shapes.
  • With more complex shapes, we need to take three steps: layout the path, stroke the path, and fill the path.
  • We can save a copy of the image we created with Canvas using a function that saves the image to a url in jpg or png format.
  • The importance of drawing an image that we already have to Canvas, is that now we can have a canvas object which we can manipulate to our liking with the features of Canvas.
  • The drawOneFrame function manipulates the image that we draw on the Canvas but it does so for each frame, so it is useful for manipulating videos, since they are images that change in each frame.
  • SVG is an image in vector format, while the other images known as jpg or png are bitmaps, therefore SVG images can be enlarged or reduced without losing quality.
  • Canvas does not create content in the DOM, SVG does create content in the DOM and you can interact with it.
  • The Drag and Drop allows to take the interaction with the user to another level more than anything with the smart mobile devices that have a touch screen that results in a very attractive combination.
  • With Drag and Drop we take one element to another "dragging" it from one side to another, being able to exchange data, and perform different tasks.
