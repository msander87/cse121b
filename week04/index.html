Questions

  • Without forms you could not have a user account on facebook, instagram, google, etc? Isn't there another way these famous websites would allow their users to login or register?
  • (document.forms[0];) Is this the best way to identify a form? Isn't it safer for a form to have a unique ID?
  • How are the focus, blur, and change methods useful?
  • Why is it important to validate the data with javascript before sending it to the server?
  • Is it advisable to use the new html attributes, considering that it may not be compatible with some browsers?
  • Is using disabled on a submit button the most recommended way to prevent submitting a form before validating it?

Notes

  • Each form control has an initial value that can be specified in the HTML code.
  • Of the options shown in chapter 8 of the book "JavaScript: Novice to Ninja, 2nd Edition", this is the safest way to call a form element: const input = form['searchInput']
  • input.value is not good, the user when clicking on the input element, the value does not disappear. The placeHolder is much more recommended.
  • The main difference between radiobuttons and check boxes is that in radiobuttons we can only select one, while in check boxes several values can be selected at once.
  • Reset button are not recommended for usability reasons.
  • Client-side validation is more for a good user experience, to help users fill out the form correctly, and to filter requests to the server. It should not be used as the main security filter for the data that is going to be entered to the server, since it is easy to manipulate the Javascript values, that is why there should also always be a good validation on the server side.
