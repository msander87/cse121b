Notes

  • Each form control has an initial value that can be specified in the HTML code.
  • Of the options shown in chapter 8 of the book "JavaScript: Novice to Ninja, 2nd Edition", this is the safest way to call a form element: const input = form['searchInput']
  • input.value is not good, the user when clicking on the input element, the value does not disappear. The placeHolder is much more recommended.
  • The main difference between radiobuttons and check boxes is that in radiobuttons we can only select one, while in check boxes several values can be selected at once.
  • Reset button are not recommended for usability reasons.
  • Client-side validation is more for a good user experience, to help users fill out the form correctly, and to filter requests to the server. It should not be used as the main security filter for the data that is going to be entered to the server, since it is easy to manipulate the Javascript values, that is why there should also always be a good validation on the server side.
  • The three main concepts in OOP: encapsulation, polymorphism and inheritance.
  • Encapsulation, it is used to create a code that cannot be modified, and only offer its functionality.
  • Polymorphism is used so that the same method can be used for several objects, being able to make it more specific for each object by overwriting that method, without the need to change everything.
  • Inheritance is used to create objects that already have functionalities and characteristics of other previous objects, being able to add other more specific characteristics.
  • Constructor function is a function that defines the properties and methods of an object.
  • It’s recommended to always use literals to create arrays
  • Static methods can only be called directly by the class and not by objects that are instances of that class.
  • The prototype is used when we want all instances of this prototype to always share the same property values. And when it has to change, it can be changed for everyone.
  • Public methods can be changed simply by assigning a new value to a property of an object. Instead, private methods can only be accessed via getter and setter functions in which we can determine the rules for property changes as we please.
  • The extendes keyword, is to inherit methods and properties to a child class. Then you can add other methods or properties specific to that class.
  • We can add new methods and features to objects that already exist. It can be very helpful depending on the program we are doing. You must call the object.prototype and add the method that you want to apply. Example: Number.prototype.isEven = Number.prototype.isEven = function() {return this%2 === 0;}. This technique is known as monkey-patching.
  • All object properties have the following attributes stored in a property descriptor: value, writable, enumerable, configurable.
  • We can see the descriptor of each property like this: Object.getOwnPropertyDescriptor(object, "property")
  • The get() and set() methods can be used to control how a property is set using assignment and the value that is returned when a property is queried
  • The getter and setter methods give you much more power in controlling the way property assignment works.
  • We can copy an object using the Object.create method: const copyObject = Object.create(the object we want to copy goes here)
  • We can use mixin() function is to add a large number of properties to an object all at once
  • In using classes, it’s recommended to they don’t have too many properties and methods
  • If we want only one method of a class it is better to borrow than to inherit all the features of that class. Since it can cause a lack of control on our part towards the code.
  • The value of this is not retained inside nested functions, which can cause errors. This can be worked around by using that = this, using the bind(this) method and using arrow functions.
  • The modules must be specific and fulfill a single task, in this way they can be totally reusable.
  • It is recommended to implement code as loosely coupled as possible, as this is more reusable for other applications or needs.
  • All code in modules is always in strict mode
  • A module has its own global scope, so the module's global variables can only be used inside this module.
  • We use "export" to indicate that something from the module can be used.
  • Having more than one default export will result in a syntax error.
