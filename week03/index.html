Notes

  • Reserved words that will not give me an error but that are good practice not to use: undefined, NaN, Infinity.
  • The main purpose of "const" is not to change a value of a variable that we don't want to be changed.
  • It is best practice and recommended to name variables that start with a lowercase letter. And that the name describes what that variable or object is going to contain.
  • Variables that start with an underscore, generally refer to private properties and methods, so it's best to not follow this convention for your own variable names.
  • Symbols are the only primitives that don't have a literal form. The only way to create them is to use the Symbol() function
  • In comparisons or conditions it is always better to use "===", since using only two == can cause some problem, since it does not differentiate in the data type.
  • The big difference between an array and a set is that in an array we can store duplicate values, unless you indicate otherwise, while in Set it is not allowed. Because an array is a collection of indices and a Set is a collection of keys.
  • Objects are limited to using strings for key values, whereas maps can use any data type as a key
  • Maps are like dictionaries in other languages like Python
  • When we want a loop to be used directly because we already know that the initial condition is valid to execute the loop, we can use "do while" instead of "while"
  • Don’t Repeat Yourself, or DRY, is a principle of programming that specifies that every part of a program should only be written once. This avoids duplication and means there’s no need to keep multiple pieces of code up to date and in sync.
  • Any parameters a function needs are set when the function is defined. When a function is invoked, it is provided with arguments.
  • Functions declaration are the same as array functions, it is a question of habit or taste. Short functions are best for use in arrow function format since they take up a single line.
