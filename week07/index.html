Notes

  • Functions are first-class objects, so they can have properties and methods themselves.
  • We can use the browser cache to save results that take a long time to process and thus avoid wasting time and processing.
  • Temporary variables are declared inside an IFFE and are used to avoid causing a crash while programming the code, since they only exist while the IFFE is being processed.
  • An IFFE is useful when we want a function and its internal variables to be executed only once.
  • Using IFFE is also recommended when we want to use strict mode, by putting strict mode in a function using IFFE we make sure that only this portion is executed that way, avoiding incompatibilities with other codes.
  • IFFE is a useful approach for separating parts of a program into discrete sections, especially for testing purposes.
  • One resource we can take advantage of when we need a function to be initialized in some way, but then changed, is to redefine the function by itself.
  • Init-Time Branching is a technique where we first check some condition such as if an object exists, and if it doesn't exist we change the function with an if and else. This way we can the first time if an object is not yet created use a second option and after the object already exists use the function as we intended it to be used with the object.
  • Callbacks are used to wait for a function to execute when an event occurs, but allow the rest of the code to continue executing while the event does not occur. (event-loop)
  • A callback is a function that is provided as an argument to another function.
  • The use of promises allows us to better organize our code, executing processes that must be fulfilled after others are fulfilled, and allowing the rest of the program to continue executing at the same time.
  • A closure is a reference to a variable that was created inside the scope of another function, but is then kept alive and used in another part of the program.
  • The difference between the yield and the return keywords is that by using yield, the state of the value returned is remembered the next time yield is called.
  • Higher-order functions are functions that accept another function as an argument, or return another function as a result, or both.
  • Functional programming involves breaking processes down into steps that can be applied as a series of functions.
  • A curried function expects that all the parameters are passed in the function, in case some parameter is not passed, the function must have a logic that does a different process to return a value consistent with the function.
