Questions

  • Is Localstorage recommended for big projects or real projects?
  • Is it a safe way to save important data?
  • In what form is the data stored and for how long?

Notes

  • Localstorage is one way we can save user input data. This data is stored indefinitely, each data must be saved with a unique variable name, a "Key value" so that we can call it later.
  • These values that we save with LocalStorage are only saved in the browser that the user is using and on their computer, nothing is sent to the server. In other words, they are only used and saved on the client side.
  • Some examples of errors that I can foresee is that since localstorage uses a single variable, or key value for a certain data, only one person can save data at a time per web page, since if another person enters another data in that variable, this variable will be updated and will have another value, unless each variable that is saved gives the user the option to choose the key value of the data, but in this case it is quite likely that errors will occur, users can choose long key values that later they cannot remember or write exactly the same as they wrote it, of course we can always do something to filter the errors, but that does not change that this It is not an optimal way to store data safely and efficiently. It's not like using a database where each person with a user can come to the computer and have their personal data for each web page.
